# Internationalization (i18n)

## Overview

Multi-language support powered by [i18next](https://www.i18next.com/), [react-i18next](https://react.i18next.com/), and [expo-localization](https://docs.expo.dev/versions/latest/sdk/localization/). The device locale is auto-detected at startup and the app falls back to the configured default locale when a translation is unavailable.

## Default Implementation

| Library | Role |
| --- | --- |
| `i18next` | Core translation engine |
| `react-i18next` | React bindings (`useTranslation`, `I18nextProvider`) |
| `expo-localization` | Device locale detection |

This is a **built-in** feature (not swappable via a service interface) -- it uses i18next directly.

## Configuration

In `src/config/starter.config.ts`:

```ts
i18n: { enabled: true, defaultLocale: 'en' }
```

- **`enabled`** -- when `false`, `I18nProvider` becomes a passthrough and `useAppTranslation` returns keys as-is.
- **`defaultLocale`** -- the fallback language code used when the device locale is not supported.

## Usage

### Translating text

```tsx
import { useAppTranslation } from '@/features/i18n';

export function MyScreen() {
  const { t } = useAppTranslation();

  return <Text>{t('common.loading')}</Text>;
}
```

### Switching languages

```tsx
import { useLanguage } from '@/features/i18n';

export function LanguageSwitcher() {
  const { language, setLanguage, availableLanguages } = useLanguage();

  return (
    <>
      {availableLanguages.map((lang) => (
        <Button key={lang} title={lang} onPress={() => setLanguage(lang)} />
      ))}
    </>
  );
}
```

## Adding a New Language

1. Create a new JSON file in `src/features/i18n/locales/` (e.g., `fr.json`) with the same key structure as `en.json`.
2. Import it in `src/features/i18n/i18n.ts` and add it to the `resources` object:
   ```ts
   import frLocale from './locales/fr.json';
   // ...
   resources: {
     en: { translation: enLocale },
     es: { translation: esLocale },
     fr: { translation: frLocale },
   },
   ```
3. Add the language code to the `AVAILABLE_LANGUAGES` array in `src/features/i18n/hooks/use-language.ts`.

## Removing the Feature

- **Disable**: set `i18n.enabled` to `false` in `starter.config.ts`. The provider becomes a passthrough and hooks return raw keys.
- **Remove entirely**: delete the `src/features/i18n/` folder, remove the `I18nProvider` from `src/lib/providers/app-providers.tsx`, and uninstall `i18next`, `react-i18next`, and `expo-localization`.
